# Excalidraw 自托管 迁移与部署文档

本文档说明如何在不同服务器上迁移和部署 my_excalidraw_cc 项目，包含所有注意事项。

---

## 一、项目结构

```
my_excalidraw_cc/
├── excalidraw/              # Excalidraw 主应用（前端 + 协作逻辑）
│   ├── excalidraw-app/      # Vite 前端应用
│   ├── packages/            # 核心包
│   ├── start.sh             # 启动前端 + 协作服务
│   ├── stop.sh              # 停止前端 + 协作服务
│   ├── ecosystem.config.js  # PM2 进程配置
│   ├── all-start.sh         # 启动前端 + 协作 + Nginx
│   ├── all-stop.sh          # 停止全部
│   ├── setup-https-selfsigned.sh   # 自签名 HTTPS 配置
│   ├── setup-https.sh       # Let's Encrypt HTTPS 配置
│   └── nginx-excalidraw.conf # Nginx 配置参考
├── excalidraw-room/         # 协作 WebSocket 服务
│   └── dist/index.js       # 需先 npm run build
├── start.sh                 # 顶层总启动
└── stop.sh                  # 顶层总停止
```

---

## 二、环境要求

| 依赖 | 版本要求 | 说明 |
|------|----------|------|
| Node.js | >= 18.0.0 | 必须 |
| npm | 建议 8+ | 项目使用 npm（无 yarn 也可） |
| PM2 | 最新版 | 进程管理器，支持自动重启 |
| Nginx | 1.18+ | 用于 HTTPS 反向代理 |
| OpenSSL | 系统自带 | 自签名证书 |
| lsof | 系统自带 | 启停脚本查找进程用 |

**注意**：不强制要求 Yarn，脚本已适配 npm。

## 三、迁移步骤

### 3.1 需要迁移的文件/目录

**必须完整拷贝：**
- `my_excalidraw_cc/` 整个文件夹（含 excalidraw 和 excalidraw-room）

**可忽略（会重新生成）：**
- `excalidraw/node_modules/`
- `excalidraw-room/node_modules/`
- `*.log`、`.excalidraw.pid`、`.proxy.pid`
- `excalidraw/.husky/`（若不需要 Git hooks）

**建议排除以加快迁移：**
- `node_modules`（在新服务器执行 `npm install`）
- `excalidraw-room/node_modules`
- `excalidraw/excalidraw-app/node_modules`

**迁移命令示例：**
```bash
# 方式 1：打包迁移（排除 node_modules）
cd /path/to
tar --exclude='node_modules' --exclude='*/node_modules' \
    -czvf my_excalidraw_cc.tar.gz my_excalidraw_cc/

# 方式 2：使用 rsync
rsync -avz --exclude='node_modules' --exclude='*/node_modules' --exclude='*.log' \
    /source/my_excalidraw_cc/ user@new-server:/target/my_excalidraw_cc/
```

### 3.2 不需要迁移的（在新服务器配置）

- Nginx 配置（每台服务器执行 setup 脚本）
- SSL 证书（每台服务器单独生成）
- `package-lock.json` / `yarn.lock` 可迁移以锁定依赖版本

## 四、新服务器部署流程

### 4.1 安装 Node.js

```bash
# Ubuntu/Debian
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 或使用 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install 20
```

### 4.2 解压/放置项目

```bash
# 若使用 tar 迁移
tar -xzvf my_excalidraw_cc.tar.gz -C /home/ubuntu/workspace/
cd /home/ubuntu/workspace/my_excalidraw_cc
```

### 4.3 安装依赖

```bash
# Excalidraw 主应用
cd excalidraw
npm install

# excalidraw-room 协作服务
cd ../excalidraw-room
npm install
npm run build
cd ..
```

**注意**：`excalidraw-room` 必须执行 `npm run build` 生成 `dist/index.js`，否则协作功能无法启动。

### 4.4 安装 PM2（进程管理）

```bash
# 全局安装 PM2
npm install -g pm2

# 设置开机自启（可选，需要 sudo 权限）
sudo env PATH=$PATH:/home/ubuntu/.nvm/versions/node/v24.13.0/bin /home/ubuntu/.nvm/versions/node/v24.13.0/lib/node_modules/pm2/bin/pm2 startup systemd -u ubuntu --hp /home/ubuntu
```

### 4.5 首次启动（仅 HTTP）

```bash
cd /home/ubuntu/workspace/my_excalidraw_cc
./start.sh
```

此时可访问：`http://服务器IP:3001/`

**注意**：
- 实时协作依赖 HTTPS（浏览器 `crypto.subtle` 要求）
- 仅 HTTP 时「开始会话」会报错，需配置 HTTPS
- 服务由 **PM2** 管理，进程崩溃会自动重启

### 4.6 使用 PM2 管理服务

#### 方式 A：仅 IP，无域名（自签名证书）

```bash
cd excalidraw
sudo ./setup-https-selfsigned.sh
```

- 访问：`https://服务器IP/`
- 首次需在浏览器点击「高级」→「继续访问」

#### 方式 B：有域名（Let's Encrypt）

```bash
# 确保域名 A 记录已解析到本机
cd excalidraw
sudo ./setup-https.sh your-domain.com
```

- 访问：`https://your-domain.com/`

### 4.8 使用 PM2 管理服务

PM2 使用 `ecosystem.config.js` 配置文件管理服务，支持自动重启和开机自启。

```bash
# 启动服务
cd /home/ubuntu/workspace/my_excalidraw_cc/excalidraw
./start.sh

# 停止服务
./stop.sh

# 查看状态
pm2 status

# 查看日志
pm2 logs

# 保存进程列表（服务器重启后自动恢复）
pm2 save
```

## 五、端口与防火墙

| 端口 | 用途 | 说明 |
|------|------|------|
| 3001 | Excalidraw 前端 | 仅本地，由 Nginx 代理对外 |
| 3002 | excalidraw-room 协作 | 仅本地，由 Nginx 代理 |
| 80 | HTTP | Nginx 监听，可重定向到 HTTPS |
| 443 | HTTPS | Nginx 监听，对外主要入口 |

**重要**：
- 防火墙/安全组需开放 **80** 和 **443**
- 3001、3002 仅需本机访问，**可不对外开放**

## 六、注意事项汇总

### 6.1 HTTPS 相关

- **实时协作必须使用 HTTPS**：浏览器 `crypto.subtle` 仅在安全上下文可用
- 自签名证书首次访问会有安全提示，需手动继续
- 使用 HTTPS 时，协作连接会经 Nginx 的 `/socket.io/` 代理，不要直连 `wss://IP:3002`

### 6.2 协作功能相关

- `excalidraw` 与 `excalidraw-room` 必须在**同一父目录**下（`start.sh` 通过 `WORKSPACE_DIR/../excalidraw-room` 查找）
- 自托管时 Firebase 持久化已禁用，协作房间不会保存到云端
- 建议使用「保存到本地」定期备份
- 多人协作时需保持页面通过 HTTPS 打开

### 6.3 目录结构

- 使用 `ecosystem.config.js` 配置文件，**无需**保持 excalidraw 与 excalidraw-room 的相对路径关系（配置文件中已指定绝对路径）：
  ```
  my_excalidraw_cc/
  ├── excalidraw/
  │   ├── ecosystem.config.js  # PM2 配置，使用绝对路径
  │   ├── excalidraw-app/      # Vite 前端应用
  │   ├── start.sh
  │   └── stop.sh
  └── excalidraw-room/
      └── dist/index.js        # 协作服务
  ```

### 6.4 无 GUI 服务器

- 脚本已设置 `BROWSER=none`、`--open false`，防止在无桌面环境下因 `xdg-open` 报错退出

### 6.5 Yarn 与 npm

- 项目默认配置为 Yarn，但脚本已改为使用 npm，**不安装 Yarn 也可运行**

### 6.6 Nginx 与端口 80/443

- 80 端口可能需要 root 权限
- 如 80 被占用，需先停止占用服务再执行 setup 脚本
- Nginx 配置路径：`/etc/nginx/sites-available/excalidraw`

### 6.7 PM2 进程管理

- 使用 **PM2** + `ecosystem.config.js` 管理前端和协作服务进程
- 进程崩溃会自动重启
- 服务器重启后自动恢复（需执行 `pm2 save`）
- 开机自启命令：
  ```bash
  sudo env PATH=$PATH:/home/ubuntu/.nvm/versions/node/v24.13.0/bin /home/ubuntu/.nvm/versions/node/v24.13.0/lib/node_modules/pm2/bin/pm2 startup systemd -u ubuntu --hp /home/ubuntu
  ```

## 七、常见问题

### Q1：协作连接失败 `wss://IP:3002 failed`

**原因**：通过 HTTPS 访问时，不应直连 3002（该端口无 SSL）。

**解决**：确保使用 `https://IP/` 访问，并已执行 `setup-https-selfsigned.sh`，协作会通过 Nginx 代理。

### Q2：`Cannot read properties of undefined (reading 'generateKey')`

**原因**：在 HTTP 非 localhost 下，`crypto.subtle` 不可用。

**解决**：改用 HTTPS 访问。

### Q3：`无法保存到后端数据库`

**原因**：自托管环境下 Firebase 持久化已关闭。

**说明**：属预期行为，不影响实时协作，建议用「保存到本地」备份。

### Q4：迁移后启动失败

- 检查 `excalidraw-room` 是否已 `npm run build`
- 检查 Node 版本 `node -v` 是否 >= 18
- 查看 `excalidraw/excalidraw.log` 和 `excalidraw-room.log`

### Q5：Nginx 重启后无法访问

```bash
sudo nginx -t           # 检查配置
sudo systemctl status nginx
sudo systemctl reload nginx
```

### Q6：访问显示 `502 Bad Gateway`

**原因**：前端服务（端口 3001）未运行。

**解决**：
```bash
cd /home/ubuntu/workspace/my_excalidraw_cc/excalidraw
./start.sh
```

**排查步骤**：
```bash
# 检查端口状态
ss -tlnp | grep -E ':(3001|3002|80)'

# 应该看到：
# 0.0.0.0:80    - Nginx
# *:3001        - Vite 前端
# *:3002        - excalidraw-room
```

### Q7：excalidraw-collab 启动失败 `EACCES: permission denied 0.0.0.0:80`

**原因**：`ecosystem.config.js` 中未正确设置 `PORT=3002`，导致服务尝试绑定端口 80（需要 root 权限）。

**解决**：确保 `ecosystem.config.js` 中设置了 `PORT: 3002`。

## 八、环境变量（可选）

| 变量 | 位置 | 说明 |
|------|------|------|
| VITE_APP_PORT | .env.development | 前端端口，默认 3001 |
| VITE_APP_WS_SERVER_URL | .env.development | 协作服务 URL，默认 localhost:3002 |
| VITE_APP_DISABLE_FIREBASE_PERSISTENCE | .env.development.local | 设为 `true` 可显式禁用 Firebase |

修改后需重启服务生效。

## 九、快速迁移检查清单

- [ ] 新机已安装 Node.js >= 18

- [ ] 已执行 `npm install`（excalidraw 与 excalidraw-room）

- [ ] 使用 `https://` 访问并测试协作
- [ ] 已执行 `pm2 save` 保存进程列表（服务器重启后自动恢复）
